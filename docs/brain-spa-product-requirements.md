# Brain Spa Product Requirements

Brain Spa is one local app for changing model behavior.

The app must help the user:

- find evidence for the behavior the user wants
- generate a dataset
- train a small adapter locally
- test the model in a real harness
- see exactly what failed
- improve the data and train again
- link a Telegram bot when the user provides a real token

The active product name is Brain Spa. Old names are only source history.

## Stack

- TypeScript UI
- Python FastAPI backend
- local runtime state in `~/.brain-spa`
- Hermes Agent as the Telegram/control layer
- Chipmunk as the Brain Spa controller
- Codex, OpenCode, Grok, and Cursor as selectable worker backends when installed

No Railway. No hosted deployment. Local macOS first.

## Main Flow

1. Evidence model finds behavior proof.
2. Data model turns evidence into dataset rows and preference pairs.
3. Training model dry-runs and fine-tunes the adapter.
4. Harness model builds environments and scores behavior.
5. User fixes evidence, data, tuning, or harness rules and repeats.

## Screens

- `/` is the loop map.
- `/evidence` shows sources and behavior evidence.
- `/datasets` generates datasets and shows handoff files.
- `/tune` dry-runs training, builds adapters, and tests adapters.
- `/test` defines environments and runs harness checks.
- `/settings` handles Telegram, Hermes readiness, worker backends, and engines.

Every visible button must run a real backend call or be removed.

## Dataset Requirements

Datasets must have explicit state:

- draft
- validated
- active
- retired
- archived

Each dataset must record:

- goal
- row count
- artifact path
- quality checks
- warnings
- known weaknesses

Checks must catch:

- duplicate prompts
- repeated answers
- template-shaped rows
- source-copy risk
- train/eval leakage risk
- missing failure labels

## Training Requirements

Brain Spa must support:

- SFT JSONL
- preference pairs
- Transformers TRL
- PEFT LoRA adapters
- MLX recipe output
- Unsloth recipe output where available
- Axolotl recipe output
- LLaMA-Factory recipe output

The app must never silently switch training backend. If a backend is missing, it must say what is missing.

The local validation build must train `HuggingFaceTB/SmolLM2-360M-Instruct` with a generated dataset and save an adapter artifact.

## Example Validation Harness

The first full validation target is a persona adapter.

The goal is a small model that answers from explicit Christian conviction instead of generic self-help.

The harness scores:

- conviction grounding
- generic phrasing
- directness

The app must test both:

- a manual answer pasted by the user
- an answer generated by the trained local adapter

If the generated answer fails, Brain Spa must show the exact failed dimension so the dataset can be improved.

## Chess Harness

Chess is the first non-chat environment.

The app must not treat Stockfish as an agent. Stockfish is an engine used by the chess environment.

The chess harness must support:

- role selection: Advisor, Player, Tutor, Hybrid
- FEN input
- legal board validation through `python-chess`
- Stockfish availability check
- explanation scoring separate from board validation

Image-to-FEN is allowed only when a real local vision path is added. Until then, the app must be honest that the current chess harness validates FEN.

## Model Registry

Models must have explicit state:

- candidate
- active
- failed
- retired
- archived

Each model must show:

- base model
- parameter count
- hardware fit
- strengths
- known failures
- lifecycle actions that are actually valid

The default small persona model is `HuggingFaceTB/SmolLM2-360M-Instruct`.

## Telegram

Telegram must be local and locked to one allowed user.

The UI must let the user add:

- bot name
- BotFather token
- allowed chat ID
- model link
- enabled flag

The backend must:

- store tokens only in `~/.brain-spa/secrets`
- restrict secret file permissions
- call Telegram `getMe` to verify a live token
- keep Hermes blocked until a token is live-verified
- reject any non-matching chat ID

## Agents

Chipmunk is the controller.

Sub-agents are goal-based, not personality-based:

- Dataset Builder
- Environment Builder
- Training Operator
- Evaluation Analyst
- Model Librarian
- Runtime Operator
- Code Worker

Each sub-agent must have a goal, allowed backends, validation rules, and a clear output.

## UI Standard

Keep the current dark tactical style.

Remove:

- repeated headers
- fake metrics
- marketing copy
- vague feature descriptions
- empty pages
- controls that do nothing

Use direct labels. Let the workflow explain the product.

## Completion Standard

Brain Spa is complete only when:

- local app starts
- TypeScript UI talks to Python backend
- 100-row dataset is generated
- local adapter is trained
- trained adapter is tested by the harness
- environment harnesses work
- Telegram blocks without a live verified token
- worker backends show real detection state
- model and dataset lifecycle buttons work
- frontend build passes
- Python tests pass
- browser QA clicks every visible control
- public GitHub repository is created
