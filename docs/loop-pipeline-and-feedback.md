# Loop Pipeline And Feedback

Brain Spa changes model behavior through a four-stage loop:

1. **Evidence** finds proof of the target behavior.
2. **Datasets** turns evidence or environment runs into training rows, preference pairs, or policy transitions.
3. **Tune** runs dry checks, trains, and records local artifacts.
4. **Test** builds environments and scores behavior in realistic worlds.

The public shell ships with one concrete reference path: **Snake Policy**. It is intentionally source-only on GitHub. Rollouts, checkpoints, eval reports, and screenshots are local runtime artifacts.

## Snake Reference Path

| Stage | Snake artifact | Local path |
|-------|----------------|------------|
| Evidence | None required for first run | Source inbox starts empty |
| Datasets | Rollout trajectories and transitions | `~/.brain-spa/artifacts/datasets/snake_rollout/` |
| Tune | Policy checkpoint and training job state | `~/.brain-spa/artifacts/training/snake_rl_validation/` |
| Test | Six-board lab, watch, play, coach, arena | `/test/snake/*` |

Snake data is generated by interacting with the environment, not by seeded text rows. A fresh clone should show:

- no policy checkpoint
- no rollout rows
- no performance history
- the Snake Test pages ready to run

## Feedback Rule

Failures should become better harness requirements or dataset requirements. Do not hide misses in screenshots or loose notes.

For future harnesses, capture:

- the world state the model saw
- the action it took
- the score/reward components
- the failure comment
- the artifact path for the run

## Repository Boundary

Commit source code, tests, docs, and small config. Do not commit:

- model weights or adapter files
- `policy.pt`, checkpoints, or generated datasets
- local runtime state under `~/.brain-spa`
- Playwright screenshots or smoke-test captures
- secrets, tokens, `.env`, or Telegram runtime files
